Приложение 3
Результаты автоматизированного тестирования

В рамках проверки работоспособности информационной системы «ИРЗ: Мониторинг и учет» выполнено автоматизированное тестирование основных функций REST API. Тесты реализованы на базе фреймворка Django в модуле `webapp/backend/apps/assets/tests.py` и охватывают тринадцать сценариев, соответствующих требованиям технического задания и разделу испытаний главы 3.

Проверка выполняется в изолированной тестовой среде: перед каждым сценарием создаются справочные данные (подразделение, локации, сотрудники, категория техники), запросы к API выполняются от имени пользователя с ролью «ИТ-инженер». Для сценариев, связанных с загрузкой фото и QR-кодов, используется отдельный каталог медиафайлов. Запуск всех сценариев одной командой:

`python manage.py test apps.assets`

По завершении в консоли выводится краткий отчёт: по одной строке на сценарий со статусом «ПРОЙДЕН», затем блок «ИТОГ: ВСЕ 13 СЦЕНАРИЕВ ПРОЙДЕНЫ» и строка «Статус: OK». Дополнительно Django выводит `Ran 13 tests` и `OK`.

**Порядок запуска тестов (для скриншота и повторной проверки)**

**Вариант 1 — локально (Windows, PowerShell)**

1. Откройте терминал и перейдите в каталог backend:
   `cd D:\Диплом\webapp\backend`
2. Активируйте виртуальное окружение (если ещё не создано — см. шаг 3):
   `.\.venv\Scripts\Activate.ps1`
3. При первом запуске установите зависимости:
   `python -m venv .venv`
   `.\.venv\Scripts\Activate.ps1`
   `pip install -r requirements.txt`
4. Чтобы в консоли корректно отображался отчёт (кириллица и символы «→»), перед тестами выполните:
   `$env:PYTHONIOENCODING='utf-8'`
   `chcp 65001`
5. Запустите все 13 сценариев (флаг `-v 0` убирает зелёные точки, остаётся только понятный отчёт):
   `python manage.py test apps.assets -v 0`
6. Сделайте скриншот: заголовок, 13 строк `[01/13] ПРОЙДЕН - ...` и блок `ИТОГ` / `Статус: OK`.

**Вариант 2 — через Docker** (если контейнер backend уже собран)

Из каталога `D:\Диплом\webapp`:

`docker compose exec backend python manage.py test apps.assets`

Если сервис называется иначе, сначала проверьте: `docker compose ps`.

**Что должно получиться:** 13 коротких строк вида `[01/13] ПРОЙДЕН - ...`, затем итог `Статус: OK`. Этот вывод вставляется в диплом как рисунок 3.9.
